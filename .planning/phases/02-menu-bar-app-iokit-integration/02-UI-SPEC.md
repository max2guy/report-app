## Spacing Scale

macOS native apps use point-based layout. The 8-point scale maps directly to SwiftUI padding/spacing values.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon-to-text gap within a device row |
| sm | 8pt | Vertical padding within each device row |
| md | 16pt | Horizontal inset for popover content edges |
| lg | 24pt | Header bottom padding; separator spacing |
| xl | 32pt | Not used in this phase |
| 2xl | 48pt | Not used in this phase |
| 3xl | 64pt | Not used in this phase |

Exceptions:
- Touch/click target minimum: 44pt height for each device row (macOS HIG accessibility)
- Popover width: fixed 280pt (measured from macOS system popovers for reference)
- Popover max height: 400pt with ScrollView when device count exceeds visible area

## Typography

All sizes in points (pt). macOS system font (San Francisco) — no custom font loading.

| Role | Size | Weight | Line Height | API |
|------|------|--------|-------------|-----|
| Header title | 13pt | semibold (600) | 1.2 | `.font(.system(size: 13, weight: .semibold))` |
| Device name | 13pt | regular (400) | 1.4 | `.font(.body)` — Dynamic Type `.body` resolves to 13pt default |
| Battery percent label | 12pt | regular (400) | 1.0 | `.font(.system(size: 12, weight: .regular))` |
| No-battery caption | 11pt | regular (400) | 1.3 | `.font(.caption)` — Dynamic Type `.caption` resolves to 11pt default |

Weight palette: regular (400) + semibold (600) only. Do not use bold (700) or any other weight.

## Color

macOS native — use NSColor semantic colors and SwiftUI Color system equivalents. Do NOT hardcode hex values; use adaptive semantic colors that respond to Light/Dark Mode automatically.

| Role | NSColor / SwiftUI Color | Usage | Ratio |
|------|------------------------|-------|-------|
| Dominant surface | `.windowBackgroundColor` / `Color(NSColor.windowBackgroundColor)` | Popover background, row backgrounds | 60% |
| Secondary surface | `.controlBackgroundColor` / `Color(NSColor.controlBackgroundColor)` | Header area, separator lines, hover states | 30% |
| Accent — interactive | `.controlAccentColor` / `Color.accentColor` | Battery progress bar fill for normal-level devices (70%+) | 10% |
| Destructive / warning | — | Not applicable in this phase (no destructive actions) | — |

### Battery Level Color Thresholds

Source: REQUIREMENTS.md UI-02, CONTEXT.md specifics. These are the only three battery state colors.

| Level | Condition | Color | SwiftUI |
|-------|-----------|-------|---------|
| High | battery >= 70% | System green | `Color.green` (`.systemGreen`) |
| Medium | 30% <= battery < 70% | System yellow/orange | `Color.yellow` (`.systemYellow`) |
| Low | battery < 30% | System red | `Color.red` (`.systemRed`) |

Note: UI-02 is formally scoped to Phase 4 per REQUIREMENTS.md traceability table. However CONTEXT.md `<specifics>` lists color thresholds as part of the Phase 2 design intent, and BATT-04 requires clear differentiation of no-battery devices. Implement battery color thresholds in Phase 2 as they are integral to the popover row design contract (D-02).

### No-Battery Device Color

Source: CONTEXT.md D-04.

| State | Color | Element |
|-------|-------|---------|
| No battery data | `.tertiaryLabelColor` / `Color(NSColor.tertiaryLabelColor)` | Device name text, icon, entire row |
| No battery caption text | `.secondaryLabelColor` / `Color(NSColor.secondaryLabelColor)` | "배터리 정보 없음" label |

Accent reserved for: battery progress bar fill on devices with high (70%+) battery only. All other interactive elements use system defaults.

## Component Inventory

### 1. Menu Bar Status Item (NSStatusItem)

**Framework:** AppKit `NSStatusItem` + `NSStatusBarButton`

| Property | Value | Source |
|----------|-------|--------|
| Display format | SF Symbol icon + battery percent text | RESEARCH.md UI-01 — Claude discretion: icon+text (not icon-only) |
| Icon | `"battery.100"` SF Symbol, updated dynamically based on lowest battery device | Claude discretion |
| Text | Lowest device battery percent, e.g. `"72%"` | UI-01 requirement |
| Length | `NSStatusItem.variableLength` | RESEARCH.md Pattern 1 |
| Action | Toggle popover on click | UI-03 |

Status item icon selection logic:
- If all devices have battery >= 70%: `"battery.100"` (or appropriate fill level variant)
- If any device has 30-70%: `"battery.50"` (or appropriate fill level variant)
- If any device has < 30%: `"battery.25"` with `.systemRed` tint
- If no devices with battery data: `"bluetooth"` (neutral state)

### 2. Popover (NSPopover + SwiftUI)

**Framework:** `NSPopover` with `NSHostingController<PopoverView>` as `contentViewController`

| Property | Value | Source |
|----------|-------|--------|
| Width | 280pt | Claude discretion — macOS system popover reference |
| Height | Auto, max 400pt (scrollable beyond) | Claude discretion |
| Behavior | `.transient` — closes on outside click | RESEARCH.md Pattern 1 |
| Preferred edge | `.minY` — popover appears below menu bar | RESEARCH.md Pattern 1 |
| Arrow | System default (shown, pointing up toward menu bar) | macOS HIG |

### 4. PopoverView — Device List

Source: CONTEXT.md D-01, D-02, D-03, D-04.

| Property | Value |
|----------|-------|
| Layout | SwiftUI `ScrollView` + `LazyVStack(spacing: 0)` |
| Sort order | Battery ascending (lowest first); nil-battery devices at bottom after all battery-known devices |
| Row height | 44pt minimum (HIG touch target) |
| Row separator | 1pt `Divider()` between rows |

### 5. DeviceRowView — Normal Device Row

Source: CONTEXT.md D-02.

Layout (left to right, vertically centered):

```
[16pt left pad] [SF Symbol 16pt] [8pt gap] [Device Name 13pt] [flexible spacer] [Progress Bar 80pt] [8pt gap] [XX% 12pt] [16pt right pad]
```

| Slot | Content | Spec |
|------|---------|------|
| Type icon | SF Symbol for device category | 16pt, foreground `.labelColor` |
| Device name | `device.name` from IOBluetooth | 13pt regular, `.labelColor`, truncated with `lineLimit(1)` |
| Battery progress bar | Custom or SwiftUI `ProgressView` | Width 80pt, height 6pt, rounded capsule, color per battery threshold |
| Battery percent | `"\(battery)%"` | 12pt regular, right-aligned, color per battery threshold |
| Row background | Transparent | Hover state: `.controlBackgroundColor` at 50% opacity via `.onHover` |

### 6. DeviceRowView — No-Battery Device Row

Source: CONTEXT.md D-04, REQUIREMENTS.md BATT-04.

Layout identical to normal row but:

| Slot | Content | Spec |
|------|---------|------|
| Type icon | SF Symbol for device category | 16pt, `.tertiaryLabelColor` |
| Device name | `device.name` | 13pt regular, `.tertiaryLabelColor` |
| Battery area | "배터리 정보 없음" text in place of progress bar + percent | 11pt caption, `.secondaryLabelColor` |

### 7. SF Symbol — Device Type Mapping

Source: CONTEXT.md D-02, RESEARCH.md Pattern 2.

| Device Type | CoD Major Class | SF Symbol | Fallback |
|-------------|-----------------|-----------|---------|
| Mouse | 0x05 + pointing minor | `"computermouse"` | `"cursorarrow"` |
| Keyboard | 0x05 + keyboard minor | `"keyboard"` | `"keyboard"` |
| Headset / Headphones | 0x04 (Audio/Video) | `"headphones"` | `"speaker.wave.2"` |
| Gamepad / Joystick | 0x08 (Toy) | `"gamecontroller"` | `"gamecontroller"` |
| Other / Unknown | any other | `"dot.radiowaves.left.and.right"` | `"dot.radiowaves.left.and.right"` |

All SF Symbols must be available on macOS 13+ (Ventura). Verify symbol availability at compile time with `NSImage(systemSymbolName:)`.

## Interaction Contract

### Menu Bar Click

| State | Behavior |
|-------|---------|
| Popover closed | `popover.show(relativeTo:of:preferredEdge:)` — opens popover |
| Popover open | `popover.performClose(nil)` — closes popover |
| Click outside popover | `.transient` behavior auto-closes — no explicit handler needed |

### Device Connection / Disconnection (DISC-03)

| Event | UI Response |
|-------|------------|
| Device connects | Append to list, re-sort, update header count, update status item text |
| Device disconnects | Remove from list, update header count, update status item text |
| Battery level changes (polling) | Update progress bar + percent label + status item text in place |

Update mechanism: `@Published` properties on `BluetoothService` observed by SwiftUI views via `@ObservedObject` or `@StateObject`.

### Battery Polling

Source: Claude discretion (CONTEXT.md). Not user-configurable in this phase (Phase 3 adds that).

| Property | Value |
|----------|-------|
| On app launch | Read immediately (no delay) |
| Polling interval | Every 60 seconds via `Timer.scheduledTimer` |
| On popover open | Trigger immediate refresh before display |
