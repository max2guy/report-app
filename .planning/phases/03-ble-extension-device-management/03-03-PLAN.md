Phase 3 전체 기능:
- BATT-02: BLE GATT Battery Service(0x180F) 배터리 읽기 (BLEService)
- BATT-03: 배터리 갱신 주기 기본 5분 (PollingInterval.fiveMin = 300)
- MGMT-01: 팝오버 각 장치 행에 모니터링 토글
- MGMT-02: 토글 상태 + 폴링 간격이 앱 재시작 후에도 유지 (UserDefaults)
- MGMT-03: 설정 창에서 폴링 간격 변경 (1/2/5/10/30분 Picker)
  </what-built>
  <how-to-verify>
1. 앱 실행:
   ```bash
   /Users/kimwoojung/report-app/.claude/worktrees/unruffled-euclid/BTBatteryMonitor/.build/debug/BTBatteryMonitor
   ```

2. **헤더 설정 버튼 확인 (MGMT-03)**
   - 메뉴바 아이콘 클릭 → 팝오버 열기
   - 헤더 오른쪽에 기어(⚙) 아이콘 버튼이 보이는가?
   - 기어 버튼 클릭 → 설정 창(NSPanel)이 열리는가?
   - 설정 창에 "배터리 갱신 주기" Picker가 있고 "5분"이 기본 선택인가?
   - 다른 간격(예: "1분")으로 변경 → 창 닫기

3. **장치 모니터링 토글 확인 (MGMT-01)**
   - 팝오버에서 장치 행 오른쪽에 토글 스위치가 보이는가?
   - 토글을 끄면 해당 장치가 목록에서 즉시 사라지는가?
   - 토글을 다시 켜면 장치가 다시 나타나는가?

4. **설정 영속 확인 (MGMT-02)**
   - 한 장치의 토글을 끈 상태로 앱 종료 (Cmd+C 또는 프로세스 킬)
   - 앱 재실행
   - 꺼둔 장치가 여전히 비활성 상태인가?
   - 폴링 간격이 "1분"으로 유지되는가?

5. **BLE 배터리 읽기 확인 (BATT-02)** — 선택 사항 (BLE Battery Service 지원 장치 필요)
   - BLE GATT 0x180F를 지원하는 장치가 연결된 경우: 배터리 레벨이 IOKit 데이터 없이도 표시되는가?
   - K3 등 HID Generic 장치가 있다면: 배터리 레벨이 표시되는가?

**예상 동작:**
- 설정 버튼 → NSPanel 창 열림 (Dock 아이콘 잠깐 표시되다 사라지는 것은 정상 — API 한계)
- 토글 → 즉시 목록에서 사라짐/나타남
- 앱 재시작 → 설정 유지됨
